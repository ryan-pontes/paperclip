# v2026.616.0

> Released: 2026-06-16

## Highlights

- **Auto-wake the active CEO on approval-pending interactions** - When an interaction lands in an approval-pending state, the active CEO is now woken automatically so board-blocking decisions get picked up without manual nudging. ([#1](https://github.com/ryan-pontes/paperclip/pull/1), NODE-134)

## Fixes

- **Clone the managed workspace when only `.paperclip` is present** - A workspace directory that contained nothing but `.paperclip` was treated as "already cloned", so the real repository was never materialized and runs stalled. The auto-clone guard now clones in that case, closing the gridlock. ([#5](https://github.com/ryan-pontes/paperclip/pull/5), NODE-141)

## CI

- **Retry flaky server vitest suites in CI** - Timing/race-sensitive server suites (heartbeat process recovery, concurrent environment dedup, concurrent companies insert) intermittently reddened the `verify_canary` / `verify_stable` gates, blocking stable promotion. Vitest now retries those suites in CI (`test.retry=2`, gated on `CI`/`GITHUB_ACTIONS`), so a genuine regression still fails every attempt while a flake passes on retry — no quarantine, full coverage preserved. Root-cause de-flake is tracked as follow-ups (NODE-147 / NODE-148 / NODE-149). ([#6](https://github.com/ryan-pontes/paperclip/pull/6), NODE-143)
